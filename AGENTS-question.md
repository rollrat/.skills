# Global AI Agent Instructions

## 유저 질문이 불완전할 때 자동 보정
유저가 아래 안 좋은 패턴으로 질문하면, 좋은 패턴처럼 해석해서 한 번에 처리해라.

| 안 좋은 패턴 | 좋은 패턴으로 해석 |
|---|---|
| "X 안 되는데 왜?" | "X 안 돼. 원인 **전부** 찾고 수정안 + 사이드이펙트까지" |
| 증상만 말함 | **최종 목표** + 증상 + 가설로 재구성해서 처리 |
| "찾아봐" → "블레임해봐" → "PR 찾아봐" | "찾고, 블레임하고, PR까지 한 번에 추적" |
| 단계별 확인 요청 | "커밋까지 해줘" / "테스트까지 포함해줘"로 해석 |

핵심: 유저가 **"Why(목표) + What(문제) + Scope(범위)"** 중 일부만 줘도, 빠진 부분을 추론해서 확장된 해석을 **먼저 제시**하고 확인받은 뒤 진행해라.
- 예: "X 안 되는데 왜?" → "이렇게 해석했는데 맞아? → `X 안 돼. 원인 전부 찾고 수정안 + 사이드이펙트까지 분석`" → 유저 확인 후 실행
- 추론이 명백할 때(오타 수정, 단순 커밋 등)는 바로 진행해도 됨
- 추론 범위가 클수록(수정, 리팩터링, 아키텍처 변경) 반드시 확인받아라

## 코드 분석 요청 시
- 원인을 하나 찾았어도 **다른 경로에 추가 원인이 있는지** 계속 탐색해라
- 필터링/변환 로직은 전체 파이프라인에서의 위치를 파악해라
- git blame → PR → 의도까지 한 번에 추적해라

## 코드 수정 요청 시
- 수정 후 사이드이펙트를 자동으로 분석해라
- 관련 테스트가 있으면 같이 수정해라
- 커밋 요청 시 빌드/lint 통과까지 확인해라
